1.2 核心更新

AI Shell 1.2 引入完整的多轮 AI Agent 自动排障模式，支持命令实时流式镜像、变更操作审批、设备厂商自动识别和 GLM 智谱大模型接入。

🤖

多轮 AI Agent 模式

基于原生函数调用（tool_calls）的多轮对话代理循环，AI 可自动规划步骤、执行只读命令、分析输出并给出下一步操作建议。

📺

命令实时流式镜像

AI 执行的只读命令实时流式输出到终端，带黄色边界标记，让你清楚看到 Agent 每一步在做什么。

🔐

命令审批卡片

涉及配置变更的命令弹出审批卡片（批准 / 拒绝 / 实验模式），通过用户 PTY 执行，确保危险操作人工确认。

🧠

GLM 智谱大模型

新增 GLM（智谱 AI）OpenAI 兼容接口支持，所有厂商（DeepSeek / GLM / OpenAI）均提供模型下拉快速切换。

🏷️

设备厂商自动识别

自动检测设备提示符（H3C/华为 <> / []、Cisco name#、Juniper user@host>），AI 系统提示包含厂商专属命令参考表。

⚡

Agent 状态栏

实时显示当前阶段（思考 / 执行中）、当前命令、目标会话和计时器，随时可一键停止。

使用流程

1.2 版本的核心体验是 AI Agent 自动排障：连接设备后，描述问题让 AI 自动规划步骤、执行检查、分析原因并给出修复方案。

01

连接设备

选择 SSH、Telnet 或串口，填写主机地址、端口、账号和串口参数，点击连接。系统自动识别设备厂商。

02

启动 AI Agent

选择 AI 厂商和模型，描述排障目标，AI 自动生成多步执行计划并开始只读命令检查。

03

审批与执行

只读命令实时流式镜像到终端；涉及变更的命令弹出审批卡片，确认后通过用户 PTY 执行。

04

获取结论

Agent 多轮分析后给出原因判断和修复建议，全程可随时点击停止按钮中断。

使用提醒

  • AI Agent 的只读命令自动执行，涉及配置变更的命令必须通过审批卡片人工确认。
  • 实验模式会批量批准后续命令，建议在测试环境使用，生产环境保持逐条审批。
  • 删除、重启、覆盖配置、修改防火墙策略、改路由等危险操作必须人工复核。
  • SFTP 当前主要用于 SSH 会话，Telnet 和串口场景以终端交互为主。
