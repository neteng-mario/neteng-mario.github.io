概述

BGP（Border Gateway Protocol）是互联网的核心路由协议，用于在不同自治系统（AS）之间交换路由信息。与 OSPF 等内部网关协议不同，BGP 是外部网关协议，设计用于处理大规模的跨域路由。

关键特性

• 路径向量协议：携带完整路径信息，便于路由选择和防止环路

• 增量更新：只发送变化的路由，而非完整路由表

• 丰富的路由策略：支持多种路径属性实现复杂路由策略

• TCP 可靠传输：使用 TCP 179 端口，确保可靠性

自治系统 (AS)

自治系统（Autonomous System，AS）是指在单一技术管理下的一组路由器，使用相同的内部路由协议和外部路由策略。AS 拥有唯一的 AS 号（ASN），用于在 BGP 路由中标识不同网络。

AS 号范围

类型 范围 说明
公有 AS 1 - 64511 可在互联网注册使用
私有 AS 64512 - 65535 仅在 AS 内部使用，不会通告到互联网

AS_PATH 属性

AS_PATH 是 BGP 的核心属性之一，记录了路由经过的所有 AS 号。当 BGP 路由器收到路由时，会检查 AS_PATH：如果发现自己的 AS 号在其中，说明存在环路，路由将被拒绝。

AS_PATH 示例：
192.168.1.0/24 via 65001 65002 65003
       ↑         ↑    ↑    ↑
       目标网络   经过AS65001、AS65002、AS65003

iBGP 与 eBGP

BGP 根据路由器所属 AS 的关系，分为 iBGP（Internal BGP）和 eBGP（External BGP）两种类型。

iBGP（内部 BGP）

• 同一 AS 内部的 BGP 会话

• 不修改 AS_PATH（AS_PATH 保持不变）

• Next_HOP 属性保持传递

• 需要全互联或使用路由反射器

eBGP（外部 BGP）

• 不同 AS 之间的 BGP 会话

• 添加自己的 AS 号到 AS_PATH 头部

• Next_HOP 通常会修改为 eBGP 邻居接口 IP

• TTL 默认为 1（可通过修改增加）

iBGP 全互联问题

在大型 AS 中，iBGP 要求所有路由器两两互联，形成全互联拓扑。这导致 n 个路由器需要 n(n-1)/2 条连接。路由反射器（Route Reflector）是解决此问题的标准方案。

路由反射器 (Route Reflector)

路由反射器是解决 iBGP 全互联问题的一种机制。通过允许路由从 RR 反射给其他 iBGP 邻居，打破了 iBGP 水平分割规则。

角色

角色 说明
Route Reflector (RR) 路由反射器，负责将学习到的路由反射给客户端
Client（客户端） 与 RR 建立 iBGP 连接的路由器
Non-Client（非客户端） 普通 iBGP 邻居，与 RR 保持全互联

反射规则

当 RR 收到来自客户端的路由时，会反射给所有其他客户端和非客户端；当收到来自非客户端的路由时，只发送给客户端。

反射器优势

• 减少 iBGP 连接数量：n 个路由器只需 n 条连接

• 降低管理复杂度

• 支持多级反射层级

BGP 路径属性

BGP 使用路径属性来描述路由特征，用于路由选择和策略控制。

属性 类型 说明
AS_PATH 公认必遵 路由经过的 AS 序列，用于防环
NEXT_HOP 公认必遵 到达目的网络的下一跳 IP 地址
ORIGIN 公认必遵 路由来源：IGP/EGP/Incomplete
LOCAL_PREF 公认自由 本地优先级，用于 AS 内部选路
MULTI_EXIT_DISC (MED) 可选非传递 告诉外部 AS 优先进入的入口
COMMUNITY 可选传递 标记路由，用于策略控制

路由传递规则

BGP 路由通告遵循以下规则：

基本传递规则

1. BGP 只会将有效路由中的最优路由通告给邻居

2. 自身产生的路由通告给所有 BGP 邻居

3. 从 iBGP 邻居收到的路由，不会传递给 iBGP 邻居（防环机制）

4. 从 iBGP 邻居收到的路由传递给 eBGP 邻居

5. 从 eBGP 邻居收到的路由传递给 iBGP 邻居

6. 从 eBGP 邻居收到的路由传递给 eBGP 邻居

防环机制

BGP 使用多种机制防止路由环路：

  • AS 内防环：iBGP 水平分割（从 iBGP 收到的路由不传递给其他 iBGP 邻居）
  • AS 间防环：AS_PATH 属性（收到包含自身 AS 号的路由时拒绝）

下一跳处理核心规则

BGP 在不同场景下的下一跳处理规则：

核心规则

• EBGP → EBGP：传给另一个 AS 时，下一跳会改为自己的接口地址

• EBGP → IBGP：传给本 AS 内部时，下一跳不变，保留原来的 EBGP 对端地址

• IBGP → IBGP：传给另一个 IBGP 邻居时，下一跳仍然不变

EBGP 传给 IBGP 后路由不生效问题

当 ASBR 从 eBGP 邻居学习到路由并传递给 iBGP 邻居时，可能会出现路由不生效的情况，主要原因是 下一跳不可达。

问题原因

• eBGP 路由的下一跳是外部 AS 的路由器地址

• 当 ASBR 将 eBGP 路由传递给 iBGP 邻居时，下一跳保持不变（遵循 EBGP → IBGP 规则）

• iBGP 邻居无法访问外部 AS 的下一跳地址，导致路由无效

解决方案（华为 VRP 系统）

在 ASBR 上对 iBGP 邻居配置 next-hop-local，强制将传递给 iBGP 邻居的路由的下一跳改为 ASBR 自身。

配置示例：
# 在同时连 eBGP + iBGP 的边界路由器上
bgp 100
 peer 10.1.1.2 as-number 100       # iBGP 邻居
 peer 10.1.1.2 next-hop-local     # 关键：传给 iBGP 时把下一跳改为自己

效果：iBGP 邻居收到的路由下一跳变为 ASBR 的地址，由于 ASBR 在同一 AS 内，下一跳可达，路由生效。

华为 VRP 系统的特殊规则

华为 VRP 系统默认情况下，BGP 不允许路由递归到默认路由 0.0.0.0/0。

重要规则：只有当路由表中存在非默认的、显式的明细路由能到达 NEXT_HOP 地址时，BGP 才会判定下一跳可达，路由生效。

影响：即使配置了默认路由，BGP 仍然需要通过明细路由来解析下一跳地址，否则路由会被视为无效。

路由选择

BGP 使用以下顺序选择最优路由（按优先级从高到低）：

优先级 条件
1 最高权重（Weight，Cisco 特有）
2 最高 LOCAL_PREF
3 本地生成的路由（Next_Hop = 0.0.0.0）
4 最短 AS_PATH
5 最低 ORIGIN 类型（IGP > EGP > Incomplete）
6 最低 MED
7 eBGP 路由优于 iBGP
8 最低 IGP 成本到 NEXT_HOP

路由优选规则

当路由器同时从不同路由协议学习到相同目的网络的路由时，会根据管理距离（Administrative Distance）选择优先级：

路由协议 管理距离 说明
直连路由 0 最高优先级
静态路由 1 手动配置
eBGP 20 外部 BGP 路由
OSPF 110 开放最短路径优先
IS-IS 115 中间系统到中间系统
RIP 120 路由信息协议
iBGP 200 内部 BGP 路由

路由优选示例

示例 1：同时收到 eBGP 路由和 OSPF 路由到同一网络时，优先选择 eBGP 路由（管理距离 20 < 110）。

示例 2：同时收到 iBGP 路由和 OSPF 路由时，优先选择 OSPF 路由（管理距离 110 < 200）。

工作流程

BGP 会话建立和路由交换的基本流程：

1. 会话建立

BGP 使用 TCP 179 端口建立连接，握手后发送 Open 消息协商参数，建立邻居关系后进入 Established 状态。

2. 路由交换

邻居关系建立后，双方交换完整路由表（通过 UPDATE 消息）。之后只有路由变化时才会发送增量更新。

3. 路由反射

当 AS 内部部署路由反射器时，客户端路由器只与 RR 建立 iBGP 连接，路由通过 RR 反射分发。

4. 跨 AS 通告

eBGP 邻居之间交换路由时，会将自己的 AS 号添加到 AS_PATH，并可能修改 NEXT_HOP 属性。
