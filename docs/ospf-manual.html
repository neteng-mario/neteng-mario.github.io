概述

OSPF（Open Shortest Path First）是一种内部网关协议（IGP），用于在单一自治系统内交换路由信息。作为链路状态协议，OSPF 维护一个完整的网络拓扑图，并使用 SPF（Dijkstra）算法计算最短路径。

关键特性

• 链路状态协议：每个路由器保存整个区域的拓扑信息

• 快速收敛：网络变化时只发送增量更新

• 无环路：SPF 算法保证路由无环

• 区域划分：支持层次化网络设计，减少 LSDB 规模

• 支持认证：支持简单密码和 MD5 认证

OSPF 区域

OSPF 使用区域（Area）来划分网络，实现层次化设计。这减少了链路状态数据库（LSDB）的规模，降低了 SPF 计算开销。

区域类型

区域类型 说明
Backbone Area (Area 0) 骨干区域，所有其他区域必须连接到 Area 0
Standard Area 普通区域，可以接收所有 LSA 类型
Stub Area 末节区域，不接收外部路由（Type 5 LSA），使用默认路由
NSSA Not So Stubby Area，类似于 Stub，但可引入少量外部路由

区域划分的优势

• 减少 LSDB 规模：路由器只需保存本区域内的链路状态

• 降低 SPF 计算开销：拓扑变化只在本区域扩散

• 限制网络波动：网络不稳定只影响局部区域

• 提高网络可扩展性

链路状态广告 (LSA)

LSA 是 OSPF 路由器之间交换的信息单元，用于描述网络拓扑。不同类型的 LSA 携带不同的网络信息。

LSA 类型

类型 名称 描述
Type 1 Router LSA 每个路由器生成，描述直连接口状态和成本
Type 2 Network LSA DR 生成，描述广播网络上所有路由器列表
Type 3 Summary LSA ABR 生成，描述区域间路由
Type 4 Summary LSA ABR 生成，描述 ASBR 位置
Type 5 External LSA ASBR 生成，描述外部路由（从其他协议引入）

LSA 头部

每个 LSA 都包含以下关键字段：

LSA 头部：
- LS Age: LSA 生存时间（秒）
- LS Type: LSA 类型（1-5）
- Link State ID: LSA 标识
- Advertising Router: 始发路由器 ID
- LS Sequence Number: 序列号，用于检测最新 LSA
- Checksum: 校验和
- Length: LSA 长度

DR 与 BDR

在广播网络（如以太网）中，如果每台路由器都与其他路由器建立 OSPF 邻居关系，会产生 n(n-1)/2 条连接。DR（Designated Router）和 BDR（Backup Designated Router）机制解决了这个问题。

路由器角色

角色 说明
DR (Designated Router) 指定路由器，作为网络的"代表"与其他路由器交换 LSA
BDR (Backup DR) 备份指定路由器，在 DR 故障时接替其职责
DROther 普通路由器，只与 DR/BDR 交换 LSA

DR 选举

DR 选举基于以下优先级（越高越优先）：

1. 最高 OSPF 接口优先级（默认 1，范围 0-255）
2. 最高 Router ID（RID）
注意：优先级为 0 的路由器不参与 DR/BDR 选举

为什么需要 DR/BDR？

• 减少邻接关系数量：从 n(n-1)/2 减少到 2n-3

• 降低 LSA 泛洪开销

• 在多路访问网络中充当 LSA 交换中心

SPF 算法

OSPF 使用 Dijkstra 算法（也称 SPF 算法）计算最短路径树，确定到每个目的网络的最佳路由。

算法原理

SPF 算法将网络视为有向图，路由器作为节点，链路作为边，每条边有权重（Cost）。算法逐步构建最短路径树：

SPF 算法步骤：
1. 初始化：已计算集合只包含根节点（本地路由器）
2. 选择：从候选集合中选择Cost最低的节点
3. 扩展：将选中节点加入已计算集合
4. 更新：更新该节点的邻居 Cost
5. 重复步骤 2-4，直到所有节点都被计算

OSPF Cost

OSPF 使用 Cost 作为路径选择的度量标准。接口 Cost = 参考带宽 / 接口带宽。

接口类型 典型 Cost
FastEthernet (100 Mbps) 1
Ethernet (10 Mbps) 10
Serial (1.544 Mbps) 64
56K Modem 1785

邻居关系

OSPF 路由器通过交换 Hello 数据包发现邻居，并建立邻接关系。邻居关系的建立经过多个状态。

邻居状态

状态 说明
Down 初始状态，未收到 Hello
Init 收到 Hello，但双向通信未建立
2-Way 双向通信建立成功，DR/BDR 选举在此发生
ExStart 交换初始状态，确定主从关系
Exchange 交换 DBD（数据库描述）数据包
Loading 加载 LSR（链路状态请求）数据包
Full 邻接关系建立完成，LSDB 同步

Hello 协议

Hello 数据包用于发现邻居、维护邻居关系。广播网络和 Point-to-Point 网络的 Hello 间隔不同。

网络类型 Hello 间隔 Dead 间隔
Broadcast / Point-to-Point 10 秒 40 秒
NBMA / Point-to-Multipoint 30 秒 120 秒

数据包类型

OSPF 使用 5 种数据包类型来建立和维护邻居关系。

类型 名称 功能
1 Hello 发现和维护邻居关系
2 DBD (Database Description) 描述本地 LSDB，交换链路状态摘要
3 LSR (Link State Request) 请求特定的 LSA
4 LSU (Link State Update) 发送请求的 LSA，包含完整的 LSA 信息
5 LSAck (Link State Acknowledgment) 确认收到的 LSA

数据包封装

OSPF 数据包直接封装在 IP 协议中，协议号为 89。不依赖 TCP 或 UDP。

OSPF 报文封装：
┌─────────┬─────────────┬──────────────────┐
│   IP    │   OSPF      │   LSA (Type 1-5)   │
│ Header  │   Header    │                   │
└─────────┴─────────────┴──────────────────┘
       ↑                        ↑
  Protocol = 89         包含路由信息
